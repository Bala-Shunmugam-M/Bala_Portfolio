What is Value Stream Mapping?
VSM traces the flow of materials and information from raw material to customer delivery. Every step is classified as either value-adding (customer pays for it), necessary non-value-adding (required but adds no customer value), or pure waste (should be eliminated). Only value-adding steps should be optimised; waste steps should be eliminated entirely.

01
Waiting (40%): The biggest waste. When one station runs faster than the next, WIP accumulates and the faster station must stop. Root cause: push scheduling with no production ceiling between stations.
02
Overproduction (25%): Each station is scheduled to run at 100% capacity regardless of downstream status. This inflates WIP and cycle time. A unit sitting in queue is not customer value — it is lead time.
03
Defects (20%): Quality inspection only at the final station means defects travel through 3–4 upstream stages before detection. A defect found late costs 5–10× more to fix than one caught at source. The rework loop also consumes production capacity.
Solution 1 — Kanban Pull System
Replace push scheduling with a Kanban pull system. Each station only produces when the downstream station signals demand via a Kanban card. This automatically caps WIP between stations to the Kanban limit, eliminating overproduction and most waiting waste simultaneously.
Solution 2 — Inline Quality Gates
Add a quality check at the output of each station (or at minimum, at the midpoint of the production flow). Stop-detect-fix at source rather than detect-rework at the end. Catch defects when they cost 1× to fix rather than 5–10× at the end of the line.
Solution 3 — WIP Cap
Set a maximum WIP limit between each pair of stations equal to 2× the takt time buffer. When the WIP cap is hit, the upstream station stops — not because of failure, but by design. This forces pace synchronisation without requiring complex scheduling software.
Why This Order Matters
Addressing Waiting before Defects would be wrong: if you first cap WIP without fixing defects, defective units immediately block the line. Fix quality first (so pull doesn't amplify defects), then introduce pull, then cap WIP. Sequence matters in Lean implementation.

Phase 1
Inline quality check at Stage 3 (midpoint) — add a visual inspection step with a clear pass/fail criterion. Document defect types for root-cause analysis.
Phase 2
Introduce Kanban between Stages 3 and 4 — start with just one interface to prove the concept and measure WIP and lead time improvement before extending to all stations.
Phase 3
Full pull system across all 5 stations — extend Kanban to all inter-station flows. Set WIP caps. Monitor takt time adherence weekly and rebalance station workloads if needed.
